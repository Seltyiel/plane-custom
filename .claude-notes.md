## Pattern di patching

- **Pattern A** (preferito): override del file CE con copia dentro
  `patches/<area>/`, applicato da `build.bat` con `copy /Y`. Il file mantiene
  la struttura del sorgente stock il più possibile e il diff è documentato
  nell'header con tag `PATCH (plane-custom) v<versione>`.
- **Full-replacement**: solo quando il diff su file CE è troppo invasivo.
  Documentare sempre nell'header e nel CHANGELOG il motivo della scelta.
- **Niente localStorage** per impostazioni: usare il backend
  (`workspace_feature_settings` JSONB toggle, v1.33e generic `useFeatureSettings`).

## Lezioni imparate da non ripetere

- **Browser cache aggressiva sui bundle Vite/React Router 7**: dopo un
  rebuild dell'immagine web il browser può servire ancora i vecchi
  chunk JS dalla cache. Sintomo: il badge è già v1.34h-X (perché
  `index.html` non è cachato e referenzia il nuovo chunk hash) ma
  alcune feature non si attivano perché un altro chunk vecchio è
  ancora in cache. Fix: **Ctrl+Shift+R** sul browser. Diagnostica:
  `grep -l '<stringa>' /usr/share/nginx/html/assets/*.js` dentro
  `plane-app-web-1` per confermare che il bundle aggiornato sia
  effettivamente sul container.
- **Diagnostica frontend Plane**: i bundle vivono in
  `/usr/share/nginx/html/assets/` dentro `plane-app-web-1` (immagine
  `plane-web-custom:latest`). Sono chunk con hash, esempio
  `root-DuJCR7vR.js`, `layout-5jD3D37T.js`. Cerca dentro questi file
  con `grep -l` per verificare che il codice patchato sia incluso
  (la minificazione preserva le stringhe letterali UI).
- **Path file plane stock**: i nomi Python con trattino del repo
  patches (es. `meeting-view.py`) vengono **rinominati con
  underscore** quando copiati in source (es. `meeting_view.py`)
  perché Python non permette trattini nei nomi modulo. Quando cerchi
  nel container API usa underscore.

- **Side effect in body di componente React = hydration mismatch (#418/#423).**
  Le `dlog()` o setState in render causano UI rotta (sidebar mancante,
  layout corrotto). Fix: usare `useEffect`. Recovery: `git stash push -u` +
  reset working dir + rebuild da ultimo commit funzionante, riapplicare
  v1.34h-1, -2, -3 uno alla volta.
- **`useMeetings(slug, undefined)` fa fetch comunque** anche se vuoi
  disabilitarlo. Per spegnerlo davvero, passare `slug=""` così la chiave
  SWR diventa null e SWR non chiama l'API.
- **Build.bat su Windows può troncare i file copiati** se la sorgente è
  aperta da un altro processo (es. editor). Sintomo:
  `display-filters-selection.tsx` da 6123→5118 byte. Workaround: `cp` da
  shell Linux montata.
- **Modifiche Python (backend) richiedono restart del container API**:
  `docker compose restart api worker beat-worker` (oppure rebuild
  dell'immagine custom se il path non è montato come volume).
